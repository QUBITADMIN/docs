## 账户类型

理解 Qubit Alpha 支持的三种跟单账户类型对于选择最适合您的方案至关重要。

### 1. API 钱包 (API Agent Wallet)

这是 Qubit Alpha 推荐的高级用法。

* **机制**：利用 Hyperliquid 的 Agent Wallet API 进行交易。
* **性质**：**非托管**。您拥有完全的资金控制权。
* **优势**：无需主账户拥有大额交易量即可创建，适合大多数用户。

### 2. Hyperliquid 子账户 (Hyperliquid Sub-account)

利用 Hyperliquid 原生的子账户功能直接导入。

* **性质**：**非托管**。
* **优势**：资金管理统一。
* **劣势**：Hyperliquid 要求主账户有 **100,000 USDC** 的总交易量才能创建子账户。

### 3. QUBIT 托管账户

* **机制**：Qubit Alpha 为您创建并管理的账户。
* **性质**：**托管**。
* **建议**：我们更推荐使用前两种非托管方案，以确保资产的绝对安全。

### 账户类型对比表

| 特性 | API 钱包 (API Agent) | Hyperliquid 子账户 | QUBIT 托管账户 |
| --- | --- | --- | --- |
| **类型** | 非托管 (推荐) | 非托管 | 托管 |
| **资金控制** | 用户完全控制 | 用户完全控制 | 平台管理 |
| **创建门槛** | 无交易量要求 | 需 Hyperliquid 交易量 (>100k) | 无 |
| **操作复杂度** | 中 | 中 | 低 |
| **适用人群** | 大多数用户 | 交易大户/专业交易员 | 新手/小额体验 |

## 关键指标

### ROI (投资回报率)

`ROI = (当前价值 - 初始价值) / 初始价值 × 100%`

### 夏普比率 (Sharpe Ratio)

* **> 1**：良好
* **> 2**：优秀
* **> 3**：卓越

### 最大回撤 (Max Drawdown)

代表了最坏情况下的潜在损失。回撤越小，策略越稳健。

### 胜率 (Win Rate)

`胜率 = 盈利交易数 / 总交易数`

### 波动率 (Volatility)

波动率越低，收益曲线越平滑。

## 跟单系统逻辑

### 多维排名 (Ranking System)

Qubit Alpha 摒弃了单一的 ROI 排名，采用多维度评分系统来寻找"聪明的钱"：

* **收益能力**：ROI、绝对收益额
* **风险控制**：回撤控制、夏普比率
* **稳定性**：交易频率、资金容量

### 投资组合级止盈止损 (Portfolio SL/TP)

* **原理**：系统监控整个子账户的净值。
* **触发**：当子账户总权益达到设定的止盈或止损线时，系统会自动平掉所有仓位并停止跟单。
* **优势**：有效防止单笔黑天鹅事件导致整个账户归零，保护您的本金。
